# Public API Reference

This document is the reference for every public xtc symbol.  The
authoritative declarations live in
[`../src/inc/xtc.h`](../src/inc/xtc.h); this document is the
human-readable counterpart, kept in sync by the discipline that
every entry here corresponds to a man page in `../man/man3/`
(asserted by `M0_CLAIMS.md` [D4]).

For background, read [`../PLAN.md`](../PLAN.md) (S)0 first.

## M0 surface

The M0 milestone exposes only the version and error-string API.
Real concurrency primitives appear in M1+.

### Library version

```c
const char *xtc_version_string(void);
int         xtc_version_components(int *major, int *minor, int *patch);
```

`xtc_version_string` returns a NUL-terminated SemVer 2.0 string
(e.g. `"0.0.1-m0"`).  The pointer is to static storage.

`xtc_version_components` writes the three numeric components into
the caller's integers and returns `XTC_OK`.  Any `NULL` argument
returns `XTC_E_INVAL` without modifying outputs.

The string and the components always agree on the leading
`MAJOR.MINOR.PATCH` triple.

See `xtc_version_string(3)`, `xtc_version_components(3)`.

### Error codes and messages

```c
typedef enum xtc_err {
    XTC_OK         =  0,
    XTC_E_INVAL    = -1,    /* invalid argument */
    XTC_E_NOMEM    = -2,    /* out of memory */
    XTC_E_NOSYS    = -3,    /* not implemented on this platform */
    XTC_E_RANGE    = -4,    /* numeric out of range */
    XTC_E_AGAIN    = -5,    /* try again later */
    XTC_E_INTERNAL = -6     /* invariant violation; bug */
} xtc_err_t;

const char *xtc_strerror(int xtc_err);
```

`xtc_strerror` returns a stable English description.  Codes outside
the known set return `"unknown"`; the function never returns NULL.

The exact message text for known codes is part of the API contract
and changes only on a major version bump.  See
[`abi-stability.md`](abi-stability.md).

See `xtc_strerror(3)`.

### Compile-time version macros

```c
XTC_VERSION_STRING   /* "0.0.1-m0" */
XTC_VERSION_MAJOR    /* 0 */
XTC_VERSION_MINOR    /* 0 */
XTC_VERSION_PATCH    /* 1 */
```

These come from `xtc_config.h`, generated by the build system from
`dist/version.in`.

## Conventions

- **Return contract.** Functions return `int` with `0` on success
  and a negative `XTC_E_*` on error.  Out-parameters are written
  only on success (the M0 version of `xtc_version_components`
  formalises this -- every other public function in xtc follows the
  same shape).
- **Static-pointer functions** (`xtc_version_string`,
  `xtc_strerror`) return pointers to immutable storage; the caller
  must not free them.
- **No `errno` reliance.**  The L0 `__os_errno` layer translates
  `errno` and `GetLastError` into `XTC_E_*` codes.  Above L0 no
  function reads `errno`.
- **No globals visible to users.**  Even the version macros are
  expanded at compile time into the consumer's translation unit;
  there is no `xtc_version` variable.

## Adding a public function

Per `M0_CLAIMS.md`:

1. Declare the function in `src/inc/xtc.h`.
2. Implement it in `src/<file>.c` with a `PUBLIC:` marker so
   `dist/s_include` picks it up.
3. Add a man page at `man/man3/<name>.3` documenting parameters
   and return value (lints in `test/m0/test_man_*.sh` enforce both).
4. Reference the function in this document.
5. Write the test before the implementation; verify it fails, then
   passes.
