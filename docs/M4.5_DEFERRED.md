# M4.5 -- fast fiber context-switch ASM (SHIPPED)

**Status:** SHIPPED.  The x86-64 SysV and Win64 MS-PE fcontext
implementations are complete and tested.  This document records the
history and remaining arch work.

## What shipped

- `src/os/asm/fctx_x86_64_sysv.S` -- Linux, *BSD, illumos
- `src/os/asm/fctx_x86_64_ms_pe.S` -- Windows x64

Both implement `make_fcontext` / `jump_fcontext` from scratch without
vendoring Boost.Context.  The implementation follows the Sys V / MS
ABI spec exactly and passes round-trip tests.

## Performance

- Before: ~600 ns per context switch (POSIX ucontext on x86-64 glibc)
- After: ~30 ns per context switch (fctx ASM)

## Remaining arch work

The following architectures still use ucontext fallback:

- `arm64-aapcs-elf` (Linux ARM64)
- `arm-eabi` (32-bit ARM)
- `ppc64le`
- `s390x`
- `riscv64`
- `sparc64`
- `mips64`
- `loongarch64`
- macOS ARM64 (`*_sysv_macho_gas.S`)

These will be added as needed when profiling shows context-switch
overhead is a bottleneck on those platforms.

## Build integration

- `XTC_HAVE_FCTX_ASM` is defined when the platform has native fctx.
- `evt/coro_uctx.c` uses fctx when available, ucontext as fallback.
- `test/m4/test_fctx.c` validates round-trip correctness.
